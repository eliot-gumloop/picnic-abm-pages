# Brand Canon — the Gumloop brand source of truth

The one place the brand lives. Agents pull from it to make and check on-brand work; the team
adds to it as the brand gets defined. It is **meant to grow** — every decision becomes an entry.

> Pair with **design-critique-rubric** when judging work (Borrow Mode / Brand Check / Gonzo Lens).

## Status convention

- 🔒 **Locked** — canonical and approved. Don't change without owner sign-off.
- 🧪 **Exploring** — current working direction, not final.
- 💡 **Proposed** — suggested, awaiting a decision.

## How this skill is organized

| File | Bucket |
|------|--------|
| `references/color.md` | Full palette, color usage hierarchy, gradient system, secondary colors for data visuals |
| `references/background-gradients.md` | Gradient crops (assets), opacity rules, 8px border-radius, halftone texture, composition |
| `references/logos-and-icons.md` | Logomark, wordmark, character icons, blob bullet icons (assets) |
| `references/motion.md` | Motion icons (dot animations), motion usage rules (assets) |
| `references/typography.md` | Typefaces, scale, usage |
| `references/voice-and-messaging.md` | Voice attributes, tone, messaging pillars |
| `references/lexicon.md` | Approved taglines, wordplay bank, terms, words to avoid |

Add more buckets as the brand grows (motion.md, iconography.md, data-viz.md). Add a row here when you do.

## How to use it

- **Need a brand fact** (a color hex, gradient rule, approved tagline, logo rule): load the relevant reference file.
- **Judging work:** combine with `design-critique-rubric`.
- Every file opens with an **entry template** — copy it to add entries consistently.

## How to contribute

1. Copy the template at the top of the relevant file and fill it in.
2. Tag a **status**, add the **date** and **who decided**.
3. **Propose-don't-overwrite** for 🔒 Locked entries.
4. Promote 🧪 → 🔒 when a decision is final.

## Governance

One owner for structural edits. Agents may suggest changes but should not silently rewrite Locked entries.
