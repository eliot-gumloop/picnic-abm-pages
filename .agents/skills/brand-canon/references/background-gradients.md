## Usage Rules

### Hierarchy
1. **Primary** — default, use unless there's a reason not to
2. **Secondary** — when Primary has been used recently
3. **Horizontal / Horizontal 2** — for banner/wide formats (3:1+ aspect ratio)
4. **Extra 1, 2, 3, Crop 2** — for variety across multi-surface campaigns

### Opacity Rules
- **60% opacity** when used behind busy data visuals, dashboards, or text blurbs — ensures content remains readable and foreground
- **70% opacity** for general backgrounds with moderate content
- **100% opacity** when used as standalone gradient assets (covers, splash pages, hero moments where gradient IS the visual)

### One Gradient Per Surface
- Limit: one gradient background per layout surface
- Never mix multiple gradient crops on the same canvas

### Halftone Texture Required
- All brand gradients must include the halftone pixel grid texture
- Smooth gradients without texture are off-brand

### Type Pairing
- White wordmark / display type sits directly on gradient
- Black body type sits on white — gradient supports from behind or alongside
- Never place long-form black type directly on gradient at 70%+ opacity

### Text Box Treatment
- All text box blurbs that sit on top of gradients use **8px border-radius** (rounded corners)
- Consistent 8px rounding across all UI cards, info boxes, data panels, and content blurbs overlaid on gradient backgrounds
- Never use sharp corners (0px) or over-rounded corners (>12px) for text containers on gradients

## Layout Templates (Gradient in Context)

### Soft Blob gradient background (from deck)
- What it is: Organic blob-shaped gradient — amber/orange center bleeding into pink/lavender/blue edges, fading to white. Halftone texture baked in.
- Use it for: Cover slides, hero moments, brand splash pages. Wordmark sits white on the gradient center.
- Limit: One per surface. Must fade to white — not boxed in a hard rectangle
- Never: Used as full-bleed on text-heavy content. Used without the halftone texture. Used with a clipped hard edge.
- Status: 🔒 · Source: Intro 2 cover slide + Moodboard (Dash Diagonal)

### Dash Diagonal gradient background (from deck)
- What it is: The same warm-to-cool gradient in a wide horizontal band format. Halftone pixel grid baked in. Named "Dash Diagonal" in the deck.
- Use it for: Structural panels within layouts (left column of split layouts, background bands). Wordmark sits white on the gradient.

- Status: 🔒 · Source: Moodboard slide (named "Dash Diagonal")

- What it is: Visible square dot grid overlaid on all gradient surfaces, variable density creating depth
- Use it for: All gradient backgrounds — baked in, never optional
- Never: Smooth gradients without texture
- Status: 🔒 · Present on all gradient surfaces in the deck

## Composition Principles

- 🔒 Gradient supports, type leads — black bold type on white is the primary content surface
- 🔒 Short copy — presentation body text is 1–2 short paragraphs max ("we keep it short and get to the point")
- 🔒 White space is structural — generous negative space on all layouts
- 🧪 One idea per surface — restraint over decoration
